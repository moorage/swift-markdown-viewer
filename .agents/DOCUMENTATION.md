  - preserving the text-only selectable surface while switching only media documents to block rendering is enough to add inline media without undoing the earlier cross-block selection work
  - `tmp/rickrolled.png` is an actual APNG despite the generic `.png` extension; the file contains `acTL`, `fcTL`, and `fdAT` chunks
  - the local macOS UI-test environment is currently blocked by a runner bootstrap failure that surfaces as a "damaged" `Swift Markdown ViewerUITests-Runner.app` dialog and an xcresult error of `Early unexpected exit ... signal kill before establishing connection`
  - the live shared model still lacks dedicated animated-image and video block kinds, so the new contract can only be enforced today as red tests
  - exposing raw markdown in one native text view fixes selection but regresses the core product expectation, so the selectable surface has to be derived from parsed blocks rather than `documentText`
  - default actor isolation on `MarkdownRenderer` was not just a warning source; once parser work moved off the main actor it could abort host-based renderer tests, so the renderer helper itself needs to opt out of `MainActor`
  - scene-appearance persistence was invalid for restoration because `restorationSession` is still `nil` until bootstrap reloads the workspace root and selected file
  - relying on app termination alone is not enough for restore durability during local development because Xcode’s Stop button can skip the normal quit-time persistence path
  - persisting session state from `onDisappear` is incorrect for quit-time behavior because disappearing windows can beat the final termination save and erase otherwise valid restore history
  - macOS temporary-directory aliases can cause workspace file enumeration to mix `/var` and `/private/var`, so raw string prefix stripping is not safe for deriving relative markdown paths
  - persisting window sessions only on scene-phase changes leaves closed-window state behind; persistence has to happen on actual session updates/removals too

  - the existing macOS UI test runner currently exits before establishing the automation connection, so active-window command behavior could not be re-proven through XCUITest in this run
  - mixed documents should segment tables as explicit blocks; treating table presence as a parser-mode switch violates compositional block semantics and causes headings to render as plaintext
  - the largest remaining CommonMark gap was the comparison oracle, not the block tree; AppKit/XML-based HTML extraction was surfacing markup internals as visible text
  - linked-image fixtures require an empty-paragraph result when preprocessing removes all visible markdown, otherwise the parser falls back to legacy raw-text handling
  - a separate human-first README is useful in this repository because the main control-plane docs are optimized for autonomous agent work, not for quickly explaining the product and the harness to a newcomer
