  - default actor isolation on `MarkdownRenderer` was not just a warning source; once parser work moved off the main actor it could abort host-based renderer tests, so the renderer helper itself needs to opt out of `MainActor`
  - scene-appearance persistence was invalid for restoration because `restorationSession` is still `nil` until bootstrap reloads the workspace root and selected file
  - relying on app termination alone is not enough for restore durability during local development because Xcode’s Stop button can skip the normal quit-time persistence path
  - persisting session state from `onDisappear` is incorrect for quit-time behavior because disappearing windows can beat the final termination save and erase otherwise valid restore history
  - macOS temporary-directory aliases can cause workspace file enumeration to mix `/var` and `/private/var`, so raw string prefix stripping is not safe for deriving relative markdown paths
